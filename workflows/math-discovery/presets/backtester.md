# Backtester

Process backend: runs the backtest spec and produces performance metrics with
statistical significance tests. The actual execution is a Python module.

## Inputs

The backtest spec path is provided via `CAAM_PURPOSE_FILE` env var. The script
reads `.plurics/shared/data/backtest-spec.json` directly.

## Execution

```bash
py -m backtester --spec .plurics/shared/data/backtest-spec.json
```

The Python module should:
1. Load the spec
2. Load the relevant OHLC data from `.plurics/shared/data/tables/`
3. Evaluate entry/exit rules bar by bar
4. Compute PnL accounting for commission and slippage
5. Run the significance tests from the spec
6. Write `backtest-result.json` per the BacktestResult schema
7. Exit 0 on success, non-zero on failure

## Output

| Path | Description |
|---|---|
| `.plurics/shared/data/backtest-result.json` | BacktestResult |
| `.plurics/shared/findings/final-report.md` | Human-readable final report |
| `.plurics/shared/data/signals/backtester.done.json` | Signal (generated by platform from exit code) |

## Metrics to Report

- **Performance**: total return, annualized return, volatility, Sharpe, Sortino, max drawdown, Calmar, win rate, profit factor, n trades
- **Per-symbol breakdown**: trades, PnL, Sharpe per symbol
- **Significance**: bootstrap CI on Sharpe, reality check, deflated Sharpe

## Verdict Categories

- **tradeable**: Sharpe > 1.0, max DD < 15%, significance p < 0.05
- **marginal**: positive but below thresholds
- **not_tradeable**: negative or not significant

Remember: a theorem with a "not_tradeable" verdict is still a valid scientific
finding. The goal is validation, not optimization.
